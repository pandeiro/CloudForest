# Prose Style\n- **Tone:** Concise and technical, focusing on clarity and directness.\n- **Audience:** Developers and system administrators managing containerized applications.\n\n# Design Principles\n- **Simplicity and Ease of Use:** Focus on a "sane-defaults" approach to minimize setup time for developers.\n- **Observability and Monitoring:** Ensure the logging stack itself is highly observable, with clear metrics and health checks for each service.\n\n# Performance Guidelines\n- **Low-Latency Ingestion:** Prioritize fast log collection and storage to ensure developers can see logs in Grafana with minimal delay.\n- **Efficient Resource Usage:** Focus on minimizing CPU and memory overhead for Promtail and Loki, especially in resource-constrained environments.\n\n# Messaging and Naming\n- **Clarity and Consistency:** Use clear and descriptive names for containers, services, and configuration parameters to improve developer experience.\n\n# Documentation and Help\n- **Self-Documenting Configuration:** Focus on well-commented configuration files for Loki, Promtail, and Grafana.\n- **Centralized Setup and Troubleshooting Guide:** Provide a comprehensive README for quick setup and common issues.\n- **Interactive Examples and Tutorials:** Offer sample configurations for common Docker containers and services.
